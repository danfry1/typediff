What it does

Structural type diffing, not string matching

Uses the TypeScript compiler to structurally compare public APIs. Doesn't trust the author's semver claim—it verifies it.

!
DETECT

Catches breaking changes

Detects breaks hidden in minor and patch bumps before they reach your CI pipeline or your users.

✓
VERIFY

Zero false positives

Uses TypeScript's own assignability checker. If typediff says it breaks, it breaks.

★
PROVEN

Battle-tested

Validated against lodash, axios, zod, TypeScript, drizzle-orm, react, vitest, date-fns, and more.

For library authors

Catch accidental breaks before you publish

Run typediff before every publish to verify your semver claim matches the actual type changes.

  • Run in CI with --exit-code to gate releases
  • Add to prepublishOnly for automatic checks
  • Compare local builds against any published version
# Compare local build against latest published
npm run build
npx typediff inspect ./dist

# Gate releases in CI
npx typediff inspect ./dist --exit-code

# Add to package.json
{
  "scripts": {
    "prepublishOnly": "typediff inspect ./dist --exit-code"
  }
}
GitHub Action

Zero-config type verification for dependency PRs

Compares type definitions for every dependency changed in the PR, posts a comment summarizing the results, and fails the check if a "minor" or "patch" bump actually contains breaking type changes. 

name: Verify Dependency Updates
on:
  pull_request:
    paths:
      - 'package.json'
      - 'package-lock.json'

jobs:
  verify-deps:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: danfry1/typediff@v1
InputDescriptionDefault
severity Minimum severity to report minor
fail-on Fail when actual exceeds claimed major
ignore Packages to skip (comma-separated globs) —
github-token Token for comments and check status ${{ github.token }}
