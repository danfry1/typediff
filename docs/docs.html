Documentation

Everything you need to use typediff — from CLI commands to the programmatic API.

Installation

Install globally or use directly with npx:

npm install -g typediff

# or run directly without installing
npx typediff inspect <pkg@old> <pkg@new>

Quick Start

Compare two versions of any npm package:

npx typediff inspect zod@3.22.0 zod@3.23.0

typediff downloads both versions, extracts their public API surfaces using the TypeScript compiler, and produces a structured diff with a semver verdict.

inspect

The primary command. Compare two package versions and get a semver verdict.

Two explicit versions

typediff inspect zod@3.22.0 zod@3.23.0

Auto-detect previous version

typediff inspect zod@3.23.0
# Automatically finds 3.22.0 and compares

Compare local build against npm

npm run build
typediff inspect ./dist my-lib@latest

Auto-detect from local package

npm run build
typediff inspect ./dist
# Reads package.json, finds the previous npm version, and compares

CI with exit codes

typediff inspect zod@3.22.0 zod@3.23.0 --exit-code
# Exit 0 = no breaking changes
# Exit 1 = breaking changes found
# Exit 2 = operational error

Filter by severity

typediff inspect zod@3.22.0 zod@3.23.0 --severity major

Ignore specific exports

typediff inspect zod@3.22.0 zod@3.23.0 --ignore "internal.*" --ignore "unstable.*"

Honor TSDoc tags

typediff inspect my-lib@1.0.0 my-lib@2.0.0 --respect-tags
# @internal/@alpha breaking changes → patch
# @beta breaking changes → minor
# @deprecated removals → minor

snapshot

Save the API surface of a package to a JSON file for later comparison. Useful for CI pipelines where you want to capture a baseline before changes.

typediff snapshot ./dist -o baseline.json

compare

Compare a previously saved snapshot against a local path or published package.

# Save a baseline
typediff snapshot ./dist -o baseline.json

# ... make changes ...

# Compare against the baseline
typediff compare baseline.json ./dist

workspaces

Scan all publishable workspace packages in a monorepo:

typediff --workspaces
typediff --workspaces --filter "packages/core-*"

All Options

Usage:
  typediff inspect <pkg@version>             (auto-detect previous version)
  typediff inspect <pkg@old> <pkg@new>
  typediff inspect ./local-path <pkg@version>
  typediff inspect ./local-path              (auto-detect from package.json)
  typediff snapshot <path> [-o <file>]
  typediff compare <snapshot.json> <path-or-pkg@version>

Options:
  --format <json|pretty>   Output format (default: auto-detect TTY)
  --severity <level>       Minimum severity: major, minor, patch
  --ignore <pattern>       Glob pattern to ignore (repeatable)
  --respect-tags           Honor @internal/@beta/@public TSDoc tags
  --verbose                Show all changes including compatible ones
  --quiet, -q              One-line verdict output
  --exit-code              Exit 1 if breaking changes found
  --include-internals      Include _-prefixed internal members
  --workspaces             Scan all workspace packages
  --filter <glob>          Filter workspaces (with --workspaces)
  --registry <url>         Custom npm registry URL (also reads .npmrc)
  --debug                  Show diagnostic debug output
  -h, --help               Show this help
  -v, --version            Show version

Configuration

Create a .typediffrc.json in your project root to set default options:

{
  "ignore": ["internal.*", "unstable.*"],
  "severity": "minor",
  "exitCode": true,
  "respectTags": true,
  "quiet": false
}

CLI flags always override config file values.

Exit Codes

CodeMeaning
0Success (or no breaking changes with --exit-code)
1Breaking changes detected (with --exit-code)
2Operational error (network failure, invalid package, etc.)

Programmatic API

Import typediff into your own tools and scripts:

import { diff, diffLocal, diffMixed } from 'typediff'

// Compare two npm versions
const result = await diff('zod', '3.22.0', '3.23.0')

console.log(result.claimedSemver) // 'minor'
console.log(result.actualSemver)  // 'major'

const breaking = result.changes.filter(c => c.semver === 'major')
// [{ path: 'ZodStringCheck', kind: 'changed', semver: 'major', ... }]

diff(name, oldVersion, newVersion, options?)

Compare two npm-published versions of a package.

const result = await diff('zod', '3.22.0', '3.23.0')
// result: ChangeSet

diffLocal(oldPath, newPath, options?)

Compare two local package directories.

const result = await diffLocal('./old-dist', './new-dist')
// result: ChangeSet

diffMixed(localPath, npmPkg, npmVersion, localIsOld, options?)

Compare a local build against a published npm version.

const result = await diffMixed('./dist', 'my-lib', '1.0.0', false)
// result: ChangeSet

extractApiTree(dtsPath, meta)

Extract the public API tree from a .d.ts file. Returns a structured representation of all exported types, functions, classes, interfaces, enums, and namespaces.

import { extractApiTree } from 'typediff'

const tree = extractApiTree('./dist/index.d.ts', { packageName: 'my-lib' })

diffApiTrees(oldTree, newTree)

Diff two API trees directly. Useful when you have already extracted the trees and want to compare them.

import { extractApiTree, diffApiTrees } from 'typediff'

const oldTree = extractApiTree('./old/index.d.ts', { packageName: 'my-lib' })
const newTree = extractApiTree('./new/index.d.ts', { packageName: 'my-lib' })
const changes = diffApiTrees(oldTree, newTree)

classifyChange(change, oldNode?, newNode?)

Classify a single change as major, minor, or patch based on the change kind and node types.

checkCompatibility(oldDts, newDts, exports)

Run TypeScript assignability checks between old and new declaration files. This is the core mechanism that eliminates false positives.

createSnapshot / loadSnapshot

Save and load API snapshots for baseline comparisons:

import { createSnapshot, loadSnapshot } from 'typediff'

// Save current API surface
const snapshot = await createSnapshot('./dist')

// Later, load and compare
const baseline = await loadSnapshot('./baseline.json')

ChangeSet

The primary result type returned by diff(), diffLocal(), and diffMixed().

interface ChangeSet {
  packageName: string
  oldVersion: string
  newVersion: string
  changes: Change[]
  actualSemver: SemverLevel    // 'major' | 'minor' | 'patch'
  claimedSemver?: SemverLevel
}

Change

Represents a single detected change between versions.

interface Change {
  kind: ChangeKind        // 'added' | 'removed' | 'changed'
  path: string            // e.g. 'ZodStringCheck' or 'ColumnBuilder.$onUpdate'
  semver: SemverLevel
  description: string
  oldSignature?: string
  newSignature?: string
}

The path field uses dot notation for nested members (e.g., ColumnBuilder.$onUpdate). The semver field reflects the severity after applying TypeScript assignability checks.

GitHub Action

Zero-config type verification for dependency PRs. The action compares type definitions for every dependency changed in the PR, posts a comment summarizing the results, and fails the check if a "minor" or "patch" bump actually contains breaking type changes.

name: Verify Dependency Updates
on:
  pull_request:
    paths:
      - 'package.json'
      - 'package-lock.json'

jobs:
  verify-deps:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: danfry1/typediff@v1

Action Inputs

InputDescriptionDefault
severity Minimum severity to report (major, minor, patch) minor
fail-on Fail when actual semver exceeds claimed (major, minor, never) major
ignore Packages to skip (comma-separated glob patterns) —
github-token GitHub token for comments and check status ${{ github.token }}

Custom Configuration

- uses: danfry1/typediff@v1
  with:
    severity: major
    fail-on: minor
    ignore: "@types/node,internal-*"

How It Works

typediff follows a 5-step pipeline to produce accurate semver verdicts:

1 Resolve Download from npm or resolve local paths
2 Extract Parse public API with the TS compiler
3 Diff Structurally diff the API trees
4 Verify TypeScript assignability checks
5 Classify Assign major / minor / patch
  1. Resolve — Downloads both package versions from npm (or resolves local paths). Supports custom registries and .npmrc configuration.
  2. Extract — Uses the TypeScript compiler to extract the public API surface: interfaces, types, functions, classes, enums, and namespaces.
  3. Diff — Structurally diffs the two API trees. Detects additions, removals, and changes at every level of the type hierarchy.
  4. Verify — Runs TypeScript's assignability checker to verify compatibility and eliminate false positives. If an old type is assignable to the new type, the change is not breaking.
  5. Classify — Each change is classified as major (breaking), minor (additive), or patch (compatible). The overall verdict is the highest severity found.

The verification step (4) is what sets typediff apart. Instead of relying on heuristics or string matching, it uses the same type checker that would fail your build. If typediff says it breaks, it will break your code.
